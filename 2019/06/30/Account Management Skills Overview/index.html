  • Advertiser like appia
    – can fire multiple parameters back to AppFlood.
  • Advertiser like AD-X
    – can fire only one parameter back to AppFlood.
  • HasOffers (MAT) tracking

  1. What URL should the advertiser provide to AppFlood?
  • Click Tracking URL (AKA: Tracking URL, Click URL, Click-through URL, etc.)
  1. What URL should AppFlood provide to the advertiser?
  • Postback URL (AKA: callback URL, install postback URL, etc.)
  1. What’s the difference between weight ON/OFF? How to decide which one to use?
  • Weight OFF: AppFlood decides whether the targeting of the install is correct or not. The advertiser will fire all install postbacks to us.
  • Weight ON: The advertiser only fires valid install postbacks to us.
  1. What is the use of Click Tracking URL and Postback URL?
  1. What’s the difference between Conversion Postback Integration and 3rd-party integration?
  • Conversion Posback Integration only needs to fire install posback to AppFlood.
  • 3rd-party integration should be able to fire d-info as well, the more d-info, the better.
  • d-info: country, language, os version
  1. What information does AppFlood need the 3rd-party tracking platform to provide?
  • d-info in postback:

&d={“version”:”{os_version}”,”country”:”{country_code}”,”language”:”{language}”}

  • Country: ISO Alpha-3 country codes (http://www.davros.org/misc/iso3166.txt).

  • Language: ISO 639-2 languages codes (http://www.loc.gov/standards/iso639-2/php/English_list.php).
